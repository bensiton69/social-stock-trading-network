# AWS EKS POC Deployment

A lean, single-user **proof of concept** deployment of the Social Stock Trading
Network API onto AWS using Kubernetes (EKS) and managed PostgreSQL (RDS),
provisioned with Terraform.

Local development is unchanged: keep using the Aspire AppHost
(`aspire run` / run `SocialStockTradingNetwork.AppHost`). Nothing in this
folder affects the local flow.

## Architecture

```
Internet -> NLB (Service type=LoadBalancer) -> EKS Deployment (API, 1 replica)
                                                     |
                                                     v
                                          RDS PostgreSQL (private subnets)
```

- **VPC**: 2 AZs, public subnets only routed to the internet (no NAT gateway).
  Worker nodes run in the public subnets; RDS sits in private subnets with no
  internet route.
- **EKS**: one managed node group, a single `t3.small` node by default.
- **RDS**: `db.t4g.micro`, single-AZ, not publicly accessible. Password is
  generated by Terraform and stored in AWS Secrets Manager.
- **ECR**: holds the API container image.
- **API**: runs EF Core migrations on startup (`RUN_MIGRATIONS=true`), exposes
  `/health` (readiness, includes a Postgres check) and `/alive` (liveness).

## Prerequisites

- [AWS CLI](https://docs.aws.amazon.com/cli/) configured with credentials
  (`aws sts get-caller-identity` should succeed).
- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.6
- [kubectl](https://kubernetes.io/docs/tasks/tools/)
- [Docker](https://docs.docker.com/get-docker/) (with buildx for `linux/amd64`)

## Deploy

```bash
# 1. Provision infrastructure (VPC, EKS, RDS, ECR). ~15-20 min on first apply.
cd deploy/terraform
terraform init
terraform apply

# 2. Build and push the API image to ECR.
cd ../..
./deploy/scripts/build-and-push.sh            # optional: pass an image tag

# 3. Configure kubectl, create the DB secret, and deploy to Kubernetes.
./deploy/scripts/deploy.sh                     # use the same tag as step 2

# 4. Grab the public endpoint and smoke test.
kubectl get svc api -n social-stocks
curl http://<EXTERNAL-HOSTNAME>/alive
curl http://<EXTERNAL-HOSTNAME>/health
curl http://<EXTERNAL-HOSTNAME>/stocks
```

The NLB hostname can take a couple of minutes to become resolvable after the
Service is created.

## Configuration

Override defaults via Terraform variables (see `terraform/variables.tf`), e.g.:

```bash
terraform apply -var="aws_region=eu-west-1" -var="node_capacity_type=SPOT"
```

## Teardown

```bash
# Delete the LoadBalancer Service first so AWS removes the NLB, otherwise the
# VPC destroy can hang on a dangling load balancer / ENIs.
kubectl delete -f deploy/k8s/service.yaml --ignore-not-found

cd deploy/terraform
terraform destroy
```

## Rough cost (us-east-1, on-demand)

| Item | Approx. monthly |
| --- | --- |
| EKS control plane | ~$73 |
| 1x t3.small node | ~$15 |
| db.t4g.micro (single-AZ) | ~$12 |
| NLB | ~$16 + traffic |
| **Total** | **~$115/mo** |

Use `terraform destroy` whenever the POC is idle to avoid charges. Setting
`node_capacity_type=SPOT` reduces node cost further.

## POC shortcuts (harden before real users)

- Worker nodes are in public subnets and there is no NAT gateway.
- The API is exposed over plain HTTP via an NLB (no TLS / custom domain).
- RDS is single-AZ with `backup_retention_period = 0` and
  `skip_final_snapshot = true`.
- Migrations run at app startup rather than as a dedicated, gated job.
- The Terraform caller is granted cluster-admin on EKS.
